# Coding Agent Workspace

Scratch space for autonomous coding tasks generated by Cornelius/Neo and executed with Milo's approval.

## Tiered Strategy

| Tier | Tool | Model |
|---|---|---|
| Critical | Claude Code (direct with John) | Claude Opus/Sonnet |
| Heavy routine (1-30 min) | ChatGPT Pro Codex agent | GPT-5.3-Codex-Spark / GPT-5.4 Pro |
| Light routine | ClawCode coding agent (cloud) | MiniMax M2.7 (Ollama Pro slot 3) |
| Fast/simple | ClawCode coding agent (local) | GPT-OSS 20B |

## Workflow

```
Neo designs architecture (ENGINEERING_BRIEF)
  → Cornelius creates execution plan (EXEC_PLAN)
    → Milo approves
      → code_execute implements in workspace/
        → Sentinel reviews output
          → Milo delivers or approves deployment
```

## Restrictions

- MILO approval required for filesystem changes outside `workspace/`
- Never modify OpenClawMaster files directly
- Never execute shell commands without plan approval
- Escalate to Codex-Spark for tasks estimated > 5 minutes
- Does NOT deploy to production (Zuck handles deployment)
- Does NOT make architectural decisions (Neo's job)

## Directory Structure

```
coding-agent/
├── workspace/     # Scratch space — generated code lands here
└── approved/      # Milo-approved code promoted from workspace
```
